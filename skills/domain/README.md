# Domain

**Practice** skills tied to a specific stack, library, or repo — *what to do and what to avoid* when working in that context (e.g. Effect TS pitfalls, a particular monorepo's conventions). The stack-specific sibling of `techniques/`.

Process skills in `global/` reference these when the work touches the relevant stack.
